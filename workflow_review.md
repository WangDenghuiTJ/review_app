# 公开示例：文档审阅工作流上线清单

更新时间：2026-04-10

## 一、目标

这是一份公开演示用的审阅文档，用来展示 Review App 的典型工作方式：

- 用 Markdown 作为正文源文件
- 用批注线程进行逐条审阅
- 必要时由“呼唤智能体”直接修改原文
- 用 `.context.md` 记录文档级背景

## 二、上线范围

本次上线只覆盖最小可用闭环：

1. 打开 Markdown 文档并进行所见即所得编辑
2. 针对选区或段落添加批注
3. 在线程中人工回复或呼唤智能体
4. 在需要时导出 PDF

暂不纳入本次公开演示范围：

- 复杂权限系统
- 多人实时协作
- 外部账号登录

## 三、建议验收点

### 阶段 A：基础链路

- 能正常加载指定 Markdown 文件
- 保存正文后可以稳定落盘
- 新建批注后能生成同名 `.comments.json`

### 阶段 B：线程处理

- 人工回复可直接写回线程
- “呼唤智能体”只处理当前线程
- 若线程明确要求改原文，智能体可直接修改正文

### 阶段 C：审阅展示

- 干净模式下显示当前正文
- 编辑审阅模式下显示相对基线的修订痕迹
- 批注列表可按状态筛选

## 四、公开仓库整理建议

- 仓库中的示例 Markdown 应替换为通用演示内容
- 示例批注不要保留真实项目背景
- README 应包含安装、启动、全局 skill 使用方式
