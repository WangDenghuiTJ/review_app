# 藻华研究下一步推进方案

更新时间：2026-04-09

## 一、基于现有笔记的判断

* 主线仍然是 `论文1 / 表层与总量矛盾`，目标不是继续扩题，而是把“表层下降不等于总量同步下降”做成可投稿的证据闭环。
* 当前已基本成立的部分是 `M01_现象证据`。这意味着“高温后表层指标下降”这条现象线已经有基础，不需要再作为近期主攻方向。
* 当前真正卡住主线的瓶颈是 `M02_反演模型`。不是因为缺想法，而是因为输入口径、样本构建规则、训练入口和验收标准都还没有完全固化。
* `M04`、`M05`、`M06`、`M07` 都依赖 `M02` 输出的表层占比 `p` 和水柱总量 `B`。只要 `M02` 不稳定，后面四个模块都会反复返工。
* 共享基础需要并行推进，但近期只做“支撑 `M02` 和后续复用”的最小闭环，不建议现在把论文2、论文3一起铺开。
* 近期默认采用“太湖先闭环，巢湖再扩展”的顺序。现有笔记显示，太湖的数据完整度和风场基础明显更高，更适合先跑出第一版稳定结果。

## 二、推荐推进顺序

1. 先把 `M02` 的输入契约和样本构建流程写死。
2. 再用太湖跑通 `M02` 单湖闭环，产出 `p` 和 `B` 的第一版稳定结果。
3. 基于 `M02` 结果快速推进 `M04` 和 `M05`，先验证主线是否成立。
4. 再做 `M06` 的独立数据补强。
5. 最后组织 `M07` 的机制闭环与写作结构。

并行支撑线：

* `S02A_GOCI-II影像下载模块`：补齐东亚影像时序，避免后续时间覆盖断档。
* `S03B_巢湖ERA5-Land气象数据下载` 与 `S03C_HRRR与URMA风场下载链路`：补齐和统一 `D02` 的变量口径。
* `S01_藻华漂移追踪方法`：做轻量 API 化和最小示例，但不抢占 `M02` 主线资源。

## 三、阶段方案

### 阶段 A：冻结 `M02` 的输入契约和执行口径

建议时长：5 到 7 天

目标：把 `M02` 从“方法框架明确”推进到“别人接手也能复现同一口径”。

本阶段必须完成的事情：

* 汇总 `D01` 和 `D02` 的全部实际输入，形成一份正式输入总表。
* 明确每个输入的路径、字段、时间覆盖、空间范围、单位和缺失情况。
* 固化 QC 规则，尤其是无效时相、缺测环境变量和异常像素的处理规则。
* 固化样本构建流程，包括像素对齐、`t_max` 选择、伪标签构建和训练集/验证集/测试集划分。
* 把当前代码入口补齐到文档中。现有笔记已经能定位到现象分析脚本和共享基础脚本，但 `M02` 的训练脚本和配置文件仍是缺口，这个缺口必须在本阶段补上。

本阶段交付物：

* 一份 `M02` 输入总表。
* 一份样本构建流程说明。
* 一份训练入口与配置清单。
* 一份验收口径说明。

本阶段结束标准：

* `M02` 的输入、样本和训练口径不再依赖口头记忆。
* 同一套规则可以稳定复跑，不会每次重训都改一遍口径。

### 阶段 B：用太湖先跑出 `M02` 第一版稳定结果

建议时长：7 到 10 天

目标：用太湖完成单湖闭环，先产出能被后续分析模块直接消费的 `p` 和 `B`。

本阶段必须完成的事情：

* 以太湖为对象，完整走通预处理、样本构建、训练、推理和结果存储流程。
* 产出表层占比 `p` 的初版结果，并检查其范围约束、空间分布和异常值。
* 产出水柱总量 `B` 的初版结果，并检查其与表层指标之间是否存在明显不合理偏离。
* 建立最小验收指标，至少包括稳定性、失败样本比例、异常案例和主要局限。

本阶段交付物：

* 太湖 `p` 初版产品。
* 太湖 `B` 初版产品。
* 一份失败案例清单。
* 一份“是否可供 `M04` / `M05` 复用”的验收记录。

本阶段结束标准：

* `M04` 和 `M05` 已经可以直接消费 `M02` 输出，而不是继续等待方法定义。

### 阶段 C：快速形成论文1的最小结果闭环

建议时长：5 到 7 天

目标：尽快回答论文1的核心问题，而不是先追求大而全的图版。

本阶段必须完成的事情：

* 在 `M04` 中先定义日尺度 `p` 的汇总方式，并产出第一版 `p-T` 关系图。
* 在 `M05` 中先定义日尺度 `B` 的汇总方式，并产出第一版 `B-T` 关系图。
* 设计一个表层-总量分离指标，直接表达高温阶段两者是否脱钩。
* 输出一组最小图组，验证“表层下降但总量未同步下降”是否已经被数据支持。

本阶段建议最先产出的图：

* `p-T` 曲线或分箱图。
* `B-T` 曲线或分箱图。
* 表层指标与总量指标的对比图。
* 一个高温阶段代表性案例图。

本阶段结束标准：

* 论文1 的主线从“现象证据”推进到“结果闭环雏形”。

### 阶段 D：补独立证据并进入写作组织

建议时长：5 到 7 天

目标：补充独立证据，并把现有结果推进到可进入论文写作的阶段。

本阶段必须完成的事情：

* 梳理 `D03` 和 `M06` 中可用于独立验证的数据，确认 Chl-a、藻密度或站点观测等数据的可用性。
* 明确反演结果与独立数据的比对方式；如果独立数据不足，也要提前定义可替代的证据链。
* 在 `M07` 中，确定正文图表结构、补充材料结构以及论文初稿提纲。

本阶段交付物：

* 一份独立验证数据清单。
* 一份验证口径说明。
* 一版论文提纲和图表结构。

本阶段结束标准：

* 论文1 已具备进入完整写作的条件。

## 四、共享基础的近期推进原则

共享基础不是暂停，而是只做主线真正需要的部分。

### 近期必须推进

`S02A_GOCI-II影像下载模块`

* 填入真实 URL 前缀。
* 运行首轮 URL 探测。
* 形成第一轮下载状态清单。
* 下载完成后接入 AFAI 处理流程。

`S03B_巢湖ERA5-Land气象数据下载`

* 启动实际下载，而不是只停留在 `dry-run`。
* 下载后复核变量、时间覆盖和空间范围。
* 把结果整理成可直接对接 `D02` 的结构。

`S03C_HRRR与URMA风场下载链路`

* **复核太湖现有风场和环境数据目录、字段和时间覆盖。**
* 固化北美 `HRRR 主驱动 + URMA 验证` 的口径。
* 统一变量矩阵、时间命名和元数据规范。

### 近期只做轻量推进

`S01_藻华漂移追踪方法`

* 先做稳定 API 或脚本入口。
* 补一份最小可调用示例。
* 近期不提升其优先级，也不急于判断是否单独发展为一篇论文。

## 五、接下来 10 天最值得立即执行的 6 件事

1. 给 `M02` 补一份正式输入总表，把 `D01`、`D02` 的路径、字段、时间范围、单位一次写清。
2. 把 `M02` 的训练脚本、配置文件、输出目录和运行入口补登记出来，消除当前“方法有了但入口缺失”的状态。
3. 用太湖、巢湖先跑一个小范围闭环样例，把像素对齐、`t_max` 选择和伪标签构建完整走通一遍。
4. 启动巢湖 `ERA5-Land` 实际下载，同时复核太湖现有环境数据是否已经满足 `M02` 输入要求。
5. 运行 `S02A` 的首轮 URL 探测和下载，形成一份可追踪的补数状态清单。
6. 在 `M04` 和 `M05` 里先定义最小图组需求，确保 `M02` 一旦出结果就能立刻转入分析。
